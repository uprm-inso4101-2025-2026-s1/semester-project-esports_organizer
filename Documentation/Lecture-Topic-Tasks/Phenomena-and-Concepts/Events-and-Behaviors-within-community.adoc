== Events and Behaviors within Community

[cols="1,1", options="header"]
|===
|Event
|Behavior

|New community is created
|A new community is registered into the collection of all communities.

|A community is removed
|A community is deleted from the collection of all communities.

|A post is added
|Add a new post by pushing the post object, giving it a unique ID.

|A post is removed
|Searches for the ID of a post and removes it. If not found, throws an error.

|A member is added
|A member’s unique ID is pushed onto the members belonging to a community.

|A member is removed
|Removes a member from the list of members in a community by finding their unique ID.

|A tag is added
|A new tag is pushed onto the tags of the community.

|A tag is removed
|A tag is removed from the community.
|===

== Summary

Identifying the events and behaviors within the Community process is essential to ensure that when it’s time to test the code, we are aware of the correct outcomes for each process.  
This table shows all important events and their behaviors within the Community process, ensuring the outcomes and behaviors are correct, helping reduce the potential chance for error and making rigorous testing easier.
